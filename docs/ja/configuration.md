## BPM Settings

遷移先: **BPM > BPM Settings**

BPMワークフロー自動化モジュールのグローバル設定です。

### 一般設定

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `enable_automation` | Check | 0 | BPM自動化のマスタースイッチ。無効の場合、ワークフロー状態の変化でBPM Actionはトリガーされません。 |
| `default_timeout` | Int | 30 | Webhookおよびn8n API呼び出しのタイムアウト（秒）。 |
| `max_retry_count` | Int | 3 | 失敗したBPM Actionの最大リトライ回数。 |
| `log_retention_days` | Int | 90 | BPM Action Logの保持日数。これより古いエントリは自動削除されます。 |

### n8n連携

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `n8n_base_url` | Data | （空） | n8nインスタンスのベースURL（例: `https://n8n.example.com`）。n8n Workflowアクションタイプに必要。 |
| `n8n_api_key` | Password | （空） | n8nインスタンスの認証用APIキー。暗号化して保存。 |

#### n8n連携のセットアップ

1. n8nインスタンスをデプロイします（セルフホストまたはクラウド）。
2. n8nの **Settings > API** でAPIキーを生成します。
3. BPM Settingsにn8nのベースURLとAPIキーを入力します。
4. `enable_automation` をチェックして自動化を有効にします。
5. アクションタイプ「n8n Workflow」でBPM Actionを作成し、ワークフローIDを指定します。

n8nのベースURLには末尾のスラッシュを含めないでください。例: `https://n8n.example.com`（`https://n8n.example.com/` ではありません）。

### BPM Actionの設定

各BPM Actionドキュメントは、ワークフロー状態の変化時にトリガーされる自動化を定義します。BPM Settingsとは別に設定します。

| フィールド | 説明 |
|-----------|------|
| Action Type | Webhook、n8n Workflow、Frappe API、Custom Script のいずれか |
| Document Type | このアクションが適用されるDocType |
| From State | 遷移前のワークフロー状態（任意の場合は空白） |
| To State | 遷移後のワークフロー状態（任意の場合は空白） |
| URL / Workflow ID | Webhook用の送信先URL、またはn8nワークフローID |
| API Method | Frappe APIメソッドのパス（Frappe APIタイプ用） |
| Script | 実行するPythonコード（Custom Scriptタイプ用） |
| Enabled | このアクションが有効かどうか |

## JP HR Settings

遷移先: **JP HR > JP HR Settings**

日本の人事労務モジュールのグローバル設定です。

### 組織設定

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `health_insurance_association` | Data | （空） | 所属する健康保険組合の名称。組合固有の料率で保険料を計算する際に使用。 |
| `pension_office_code` | Data | （空） | 管轄の年金事務所コード。 |

### 保険料計算

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `employment_insurance_rate_type` | Select | General | 雇用保険料率の業種区分。選択肢: **General**（一般）、**Construction**（建設業）、**Agriculture**（農林水産業）。 |
| `auto_calculate_premiums` | Check | 1 | 有効の場合、標準報酬月額の変更時に社会保険料が自動再計算されます。 |

### 会計年度

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `fiscal_year_start` | Select | April | 会計年度の開始月。選択肢: **April**（4月）または **January**（1月）。日本のほとんどの企業は4月を使用。 |

### 補足

- `employment_insurance_rate_type` は労働保険料率テーブルのどの列を使用するかを決定します。ほとんどの企業には「General（一般）」が適用されます。
- `health_insurance_association` は正確な保険料計算に必要です。空白の場合、協会けんぽのデフォルト料率を使用します。
- `auto_calculate_premiums` の変更は、既存の記録に遡及して適用されません。

## JP Invoice Settings

遷移先: **JP Accounting > JP Invoice Settings**

適格請求書等保存方式（インボイス制度）と電子インボイスの会社レベル設定です。

### 法人登録情報

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `company_registration_number` | Data | （空） | 国税庁が付与する13桁の法人番号。 |
| `qualified_invoice_issuer_number` | Data | （空） | 適格請求書発行事業者の登録番号。形式: **T** + 13桁（例: T1234567890123）。 |

### 税率

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `default_tax_rate` | Percent | 10 | 請求書に適用する標準消費税率。 |
| `reduced_tax_rate` | Percent | 8 | 対象品目（飲食料品、新聞）に適用する軽減税率。 |

### 電子インボイス

| フィールド | タイプ | デフォルト | 説明 |
|-----------|--------|----------|------|
| `enable_e_invoice` | Check | 0 | 電子インボイスの生成・処理を有効にする。 |
| `e_invoice_format` | Select | （空） | 使用する電子インボイスの標準規格。選択肢: **Peppol BIS**（日本で採用された欧州標準）、**JP PINT**（日本固有のPeppol対応規格）。 |

### 適格請求書等保存方式（インボイス制度）について

2023年10月に施行された適格請求書等保存方式（インボイス制度）は、消費税の仕入税額控除を受けるために、登録事業者が特定の情報を記載した請求書を発行することを義務付けています。要点:

- 適格請求書発行事業者として登録された事業者のみが、買い手が仕入税額控除を受けられる請求書を発行できます。
- 登録番号（T + 13桁）はすべての適格請求書に記載する必要があります。
- 標準税率（10%）と軽減税率（8%）は請求書上で区分して記載する必要があります。
- `qualified_invoice_issuer_number` は国税庁の公表サイトで確認できます。

## ロールの割り当て

BPMモジュールは10のロールをインストールします。**設定 > ユーザー** の「ロール」タブでユーザーに割り当ててください。

### BPM承認ロール

| ロール | 用途 | 一般的な割り当て先 |
|-------|------|-------------------|
| CRM Approver | リード・商談の承認・評価 | 営業チームリーダー |
| Approval Manager | 500万円以下のドキュメント承認 | 部門マネージャー |
| Approval Director | 2,000万円以下のドキュメント承認 | 事業部長 |
| Approval Executive | 2,000万円超のドキュメント承認 | 経営幹部（役員） |
| Budget Controller | 発注の予算確認 | 経理・財務担当 |
| Ringi Approver | 稟議書の承認 | 組織により異なる |
| General Affairs | 押印依頼・汎用申請の処理 | 総務担当 |
| Legal Reviewer | 押印依頼ワークフローの法務レビュー | 法務部門 |

### JP HRロール

| ロール | 用途 | 一般的な割り当て先 |
|-------|------|-------------------|
| HR Manager | JP HRモジュールへの全アクセス（マイナンバー含む） | 人事部門の責任者 |
| HR User | JP HR機能への標準アクセス（マイナンバーアクセス不可） | 人事スタッフ |

### 割り当てガイドライン

- ユーザーは複数のロールを持つことができます（例: 部長がApproval ManagerとApproval Directorの両方を保持）。
- 承認ロールにより、ユーザーが実行できるワークフロー遷移が決まります。適切なロールがない場合、承認/却下ボタンは表示されません。
- マイナンバーへのアクセスは、番号法に準拠するためHR ManagerとSystem Managerに制限されています。

## ワークフローのカスタマイズ

プリインストールされたワークフローは、組織の要件に合わせてカスタマイズできます。

### 承認閾値の変更

1. **設定 > Authorization Rule** を開きます。
2. 対象のDocType（例: Quotation）のルールを探します。
3. `above_value` と `approving_role` フィールドを調整します。
4. ルールを保存します。

例: 見積書のDirector承認を500万円超ではなく300万円超に変更する場合、Approval Managerルールの `above_value` を 3000000 に変更します。

### ワークフローステップの追加

1. **設定 > Workflow** で変更するワークフローを選択します。
2. ワークフローステートテーブルに新しいステートを追加します。
3. ワークフロートランジションテーブルにステート間の遷移を追加し、必要なロールとアクションを指定します。
4. 保存し、サンプルドキュメントでワークフローをテストします。

### ベストプラクティス

- **本番環境に適用する前に開発環境でテスト**してください。ワークフローの変更は即座に反映され、処理中のドキュメントに影響する可能性があります。
- 既存のドキュメントが参照している**ワークフローステートを削除しない**でください。先に処理中のドキュメントを有効なステートに変更してください。
- アプリ更新時にカスタマイズが保持されるよう**変更内容を記録**してください。カスタムワークフローは `bench migrate` 時のフィクスチャ更新で上書きされる可能性があります。
- フィクスチャの上書きを防ぐには、変更したワークフローを `hooks.py` のフィクスチャリストから除外するか、Frappeの `fixtures` 設定で特定のレコードを除外してください。
